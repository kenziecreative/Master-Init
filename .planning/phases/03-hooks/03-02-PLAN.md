<task type="auto">
  <name>Task 1: Create SessionStart and SessionEnd hooks</name>
  <files>scaffold/hooks/session-start.sh, scaffold/hooks/session-end.sh</files>
  <action>
**Create `scaffold/hooks/session-start.sh`:**

1. Source hook-utils.sh and add ERR trap (same pattern as Plan 01 hooks)
2. Read the matcher from the hook environment. Claude Code passes the matcher as the first argument or via environment variable. Check `$1` for the matcher value.
3. **If matcher is "compact", exit 0 immediately** — PostCompact handles compaction orientation to avoid duplicates. Add a comment explaining why.
4. If STATE.md does not exist (`! -f "$PROJECT_DIR/.planning/STATE.md"`), exit 0 silently
5. Extract and output orientation using read_state_field():
   - Current Position
   - Last Activity
   - Session Continuity (which contains "What a new session needs to know")
6. Format output (~8-12 lines):
   ```
   === SESSION START ===
   Position: {current position}
   Last activity: {last activity}

   What to know:
   {session continuity content}
   ====================
   ```
7. Always exit 0. Make executable.

**Create `scaffold/hooks/session-end.sh`:**

1. Source hook-utils.sh and add ERR trap
2. Read matcher from `$1` (captures exit type: the 6 stop matchers)
3. If `.planning/` directory does not exist, exit 0 silently (not a scaffolded project)
4. Build a log entry with:
   - Timestamp: `date -u +"%Y-%m-%dT%H:%M:%SZ"`
   - Matcher/exit type: `$1` (or "unknown" if empty)
   - Session duration: If a start timestamp can be calculated (check if session-log.md has a recent "start" entry from SessionStart, or if STATE.md has a last session timestamp), calculate duration. Otherwise note "duration unavailable". Keep this simple — an approximation is fine, "unknown" is acceptable per Claude's Discretion.
5. Append entry to `$PROJECT_DIR/.planning/session-log.md`. Create the file with a header if it does not exist:
   ```markdown
   # Session Log

   | Timestamp | Type | Duration |
   |-----------|------|----------|
   ```
   Append a new table row: `| {timestamp} | {matcher} | {duration} |`
6. **Cap at 100 entries:** After appending, count table rows (lines matching `^|` that are not the header). If more than 100, trim the oldest entries (remove lines from the top of the table body until 100 remain). Use tail or sed for this.
7. Always exit 0. Make executable.
  </action>

  </verify>
  <done>SessionStart outputs orientation for startup/resume/clear, is no-op for compact. SessionEnd appends structured entry to session-log.md with 100-entry cap. Both use hook-utils.sh, ERR trap, exit 0.</done>
