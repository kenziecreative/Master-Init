<task type="auto">
  <name>Task 1: Create hook-utils.sh shared utility library</name>
  <files>scaffold/hooks/hook-utils.sh</files>
  <action>
Create `scaffold/hooks/hook-utils.sh` — a sourceable bash library that all hooks will use. Include:

1. **PROJECT_DIR resolution** at the top (shared by all hooks):
   ```
   PROJECT_DIR="${CLAUDE_PROJECT_DIR:-$(pwd)}"
   ```
   Note: Do NOT use `git rev-parse --show-toplevel` in the default — that would fail in non-git projects. Use pwd as fallback.

2. **`log_error()`** function:
   - Takes two args: hook_name and error_message
   - Appends a timestamped entry to `$PROJECT_DIR/.claude/hook-errors.log` in format: `[ISO-timestamp] [hook_name] error_message`
   - Also emits to stderr: `echo "HOOK ERROR [$hook_name]: $error_message" >&2`
   - Creates .claude/ directory if it does not exist (mkdir -p)

3. **`is_git_project()`** function:
   - Returns 0 (true) if `$PROJECT_DIR/.git` exists (use `-d` test)
   - Returns 1 (false) otherwise
   - No output — pure boolean for use in conditionals

4. **`read_state_field()`** function:
   - Takes one arg: field_name (a markdown heading like "Current Position", "Last Activity", "Open Items", "Session Continuity")
   - Reads `$PROJECT_DIR/.planning/STATE.md`
   - Extracts the content under the matching `## field_name` heading up to the next `## ` heading (or EOF)
   - Outputs the extracted lines to stdout (trimmed of leading/trailing blank lines)
   - Returns 1 if STATE.md does not exist or field not found
   - Implementation: Use sed or awk. Example approach with sed: `/^## field_name$/,/^## /{ /^## /!p; }` then trim the last boundary line

5. **Trap wrapper pattern** — include a comment block showing the recommended trap usage:
   ```bash
   # Usage in hook scripts:
   #   source "$(dirname "$0")/hook-utils.sh"
   #   trap 'log_error "hook-name" "$BASH_COMMAND failed (line $LINENO)"; exit 0' ERR
   ```

Make the file NOT executable (it's sourced, not run directly). Add a header comment explaining its purpose.

  <action>
**Rewrite `scaffold/hooks/pre-compact-check.sh`** — upgrade from warning-only to active save:

1. Source hook-utils.sh: `source "$(dirname "$0")/hook-utils.sh"`
2. Add ERR trap: `trap 'log_error "pre-compact-check" "$BASH_COMMAND failed (line $LINENO)"; exit 0' ERR`
3. If STATE.md does not exist (`! -f "$PROJECT_DIR/.planning/STATE.md"`), exit 0 silently (no-op before scaffolding)
4. Build a snapshot block with:
   - Timestamp: `date -u +"%Y-%m-%dT%H:%M:%SZ"`
   - Modified files: Only if `is_git_project` returns true — use `git -C "$PROJECT_DIR" diff --name-only HEAD 2>/dev/null` and `git -C "$PROJECT_DIR" diff --cached --name-only 2>/dev/null`. If not a git project, note "non-git project — file tracking unavailable"
   - Recent commits (last 3): Only if `is_git_project` — use `git -C "$PROJECT_DIR" log --oneline -3 2>/dev/null`. Skip if not git.
5. Append the snapshot to STATE.md's Session Continuity section. Use a clear delimiter:
   ```
   ### Pre-Compaction Snapshot (auto-generated)
   - **Timestamp:** {timestamp}
   - **Modified files:** {list or "non-git project"}
   - **Recent commits:** {list or "non-git project"}
   ```
   Append at the end of the file (before the version marker comment if present). Use sed to insert before `<!-- knzinit` line, or just append if no version marker found.
6. Output to stdout: `PRE-COMPACT: State snapshot saved to STATE.md`
7. Always exit 0

**Create `scaffold/hooks/post-compact-orientation.sh`** — new PostCompact hook:

1. Source hook-utils.sh: `source "$(dirname "$0")/hook-utils.sh"`
2. Add ERR trap: `trap 'log_error "post-compact-orientation" "$BASH_COMMAND failed (line $LINENO)"; exit 0' ERR`
3. If STATE.md does not exist, exit 0 silently
4. Read and output orientation (~10-15 lines). Use read_state_field() to extract:
   - Current Position
   - Last Activity
   - Open Items
   - "What a new session needs to know" (from Session Continuity — this is a `### ` subsection, so read_state_field on "Session Continuity" and then extract the subsection content)
5. Format output as a clear orientation block:
   ```
   === POST-COMPACTION ORIENTATION ===
   Position: {current position text}
   Last activity: {last activity text}

   Open items:
   {open items text}

   Context:
   {what a new session needs to know text}
   ================================
   ```
6. Always exit 0

Make both scripts executable (`chmod +x` — note this in the action, executor should do it).
