  <name>Task 1: Create path resolution helper with CLAUDE_PLUGIN_ROOT fallback</name>
  <files>scaffold/resolve-root.sh</files>
  <action>
Create scaffold/resolve-root.sh — a bash script that resolves the plugin root directory and exports it as KNZINIT_ROOT plus reads the version from plugin.json.

Logic:
1. If CLAUDE_PLUGIN_ROOT is set and the directory exists, use it
2. Otherwise, derive dynamically: walk up from the script's own location (`dirname` of `$0` or `BASH_SOURCE[0]`) to find the directory containing `.claude-plugin/plugin.json`
3. If neither works, try known plugin directories: `~/.claude/plugins/knzinit/`, `$CLAUDE_PROJECT_DIR/.claude/plugins/knzinit/`
4. If all fail, print error to stderr and exit 1
5. Export KNZINIT_ROOT with the resolved path
6. Read version from `$KNZINIT_ROOT/.claude-plugin/plugin.json` using a simple grep/sed (no jq dependency). Export as KNZINIT_VERSION
7. Export a function `knzinit_version_marker()` that returns `<!-- knzinit v${KNZINIT_VERSION} -->`

The script should be sourceable (`. resolve-root.sh`) not just executable. Use POSIX-compatible shell where possible but bash is acceptable since the existing hooks use bash.

Include a header comment explaining usage: "Source this file to get KNZINIT_ROOT and KNZINIT_VERSION exports."
  </action>

  </verify>
  <done>resolve-root.sh exists, is sourceable, exports KNZINIT_ROOT pointing to the plugin directory, exports KNZINIT_VERSION read from plugin.json, and works both when CLAUDE_PLUGIN_ROOT is set and when it is unset (fallback path).</done>
